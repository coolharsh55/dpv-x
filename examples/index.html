This document lists the examples that provide code in a specific format intended to illustrate or describe the application of [[[DPV]]] concepts towards a particular use-case or goal.

Contributing to the DPV and its extensions The DPVCG welcomes participation regarding the DPV, including expansion or refinement of its terms, addressing open issues, and welcomes suggestions on their resolution or mitigation.

For contributions to the DPV, please see the section on GitHub. The current list of open issues and their discussions to date can be found at GitHub issues.

skos:definitionAn Example provides code in a specific format intended to illustrate or describe the application of DPV concepts towards a particular use-case or goal.
  1. An Example MUST have a title (provided using dct:title)
  2. An Example MUST have a description (provided using dct:description)
  3. An Example MUST have an identifier with prefix 'EX' (provided using dct:identifier)
  4. An Example MUST have EXACTLY 1 reference to a DPV serialization (e.g. DPV, DPV-SKOS, DPV-OWL) it uses using dct:requires)
  5. An Example MUST indicate its format (e.g. Turtle, XML) using dct:format
  6. An Example MAY have one or more contributors (specified using dct:contributor)
  7. An Example MAY have a date (e.g. creation or modification) (specified using dct:date)
  8. An Example MAY specify the source of its information (using dct:source)
  9. An Example MAY specify its primary subject or concept (using dct:subject)
  10. An Example MAY specify relevant requirements derived from it (using dct:references)
